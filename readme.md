
## video2webp.py
### 视频转换为webp图片或动图
### 依赖安装

```bash
pip install PySide6 opencv-python pillow
```

### 使用方式

```bash
python video2webp.py
```

选择视频后指定输出文件夹,并修改图片前缀,图片会依序命名.


## assign_games.py
### 任务分配器（游戏光碟共享问题）
### 场景描述
假设你有几个朋友聚在一起玩游戏机，但有多个游戏光碟，每个人都想玩遍所有游戏。每个游戏一次只能被一个人独占（插入主机），一旦开始就必须玩完才能换人。如果安排不当，就可能出现两个人同时想玩同一个游戏而必须等待的情况，导致整体时间延长。

本程序旨在为每个人规划一个游戏顺序，使得所有人在玩完所有游戏的过程中，总的完成时间（makespan）最小，从而减少等待，让大家尽快开始下一项活动。

例如，有两个人和四个游戏，程序会输出类似：

0: [游戏A, 游戏D, 游戏C, 游戏B]  

1: [游戏C, 游戏B, 游戏A, 游戏D]  

按照此顺序执行，可以有效错开长游戏和短游戏，避免资源冲突。

使用方法
命令行格式
```bash
python assign_tasks.py "任务名1,任务名2,..." "时长1,时长2,..." [人数]
```
任务名列表：用英文逗号分隔的游戏名称，可以包含中文，整体建议用双引号包裹。

时长列表：与任务名一一对应的时长（单位：小时，可以是小数），同样用逗号分隔。

人数：可选参数，默认为 2。

示例
```bash
python assign_tasks.py "火焰纹章,忍者龙剑传,超级马里奥,塞尔达传说" "32,10.5,15,20" 3
```
这表示有 3 个人，要玩 4 个游戏，每个游戏的时长分别为 32 小时、10.5 小时、15 小时、20 小时。程序将计算出每个人的最佳游戏顺序并输出。

输出示例：

```text
0: ['火焰纹章', '塞尔达传说', '超级马里奥', '忍者龙剑传']
1: ['超级马里奥', '忍者龙剑传', '火焰纹章', '塞尔达传说']
2: ['忍者龙剑传', '火焰纹章', '塞尔达传说', '超级马里奥']
```
### 注意事项
任务名和时长数量必须相等。

时长应为正数。

程序运行需要 Python 3.6 或更高版本（无需第三方库）。

结果基于模拟退火算法，不保证全局最优，但在合理时间内能给出近似最优解。

### 算法原理
程序采用模拟退火搜索最佳顺序：

用离散事件模拟评估每个方案的总完成时间。

从启发式初始解（长任务尽量错开）开始，并尝试几个随机解择优。

通过交换一个人的两个任务来生成新解，以一定概率接受较差解，避免陷入局部最优。

逐步降温，最终输出找到的最优顺序。

对于几十个任务和十人以内规模，通常能在几秒内给出结果。
